# GitHub Practice Evidence — [นายกิตติศักดิ์ นะนุนาค]

## What I did
ได้ฝึกปฏิบัติการใช้งาน Git และ GitHub ผ่านระบบ Command Line (Terminal) โดยเรียนรู้และใช้คำสั่งพื้นฐานในการควบคุมระบบปฏิบัติการ 
cd (Change Directory) เพื่อย้ายตำแหน่งเข้าไปยังโฟลเดอร์โปรเจกต์
touch เพื่อสร้างไฟล์หลักฐาน .md ใหม่ขึ้นมาในระบบ
nano เพื่อเปิดโปรแกรมแก้ไขข้อความใน Terminal และเขียนเนื้อหารายงานลงในไฟล์

## My role in project
Role: Hacker (Developer / Programmer ประจําทีม) รับผิดชอบในส่วนของการจัดการโครงสร้างไฟล์ แก้ไขโค้ด และเขียนเอกสารประกอบโปรเจกต์ผ่านคอมมานด์ไลน์

## Reflection
วันนี้ฉันเรียนรู้อะไรจากการใช้ GitHub: เรียนรู้วิธีการทำงานกับระบบไฟล์ผ่าน Terminal ตั้งแต่การเดินทางเข้าโฟลเดอร์ การใช้คำสั่งสร้างไฟล์ และเขียนข้อความลงในไฟล์เนื้อหา .md โดยไม่ต้องพึ่งพาหน้าต่าง UI ปกติ รวมถึงเข้าใจภาพรวมของการส่งไฟล์กลับไปยังคลังหลักของอาจารย์

ฉันติดปัญหาอะไร: ในตอนเริ่มต้นโปรเจกต์และดึงคลังลงมา ไม่พบโฟลเดอร์ docs หรือโครงสร้างที่ถูกต้องในเครื่อง ทำให้ไม่สามารถระบุตำแหน่งที่จะวางไฟล์หลักฐานได้

ฉันแก้ปัญหาอย่างไร: ตัดสินใจติดต่อและสอบถามอาจารย์ผู้สอนโดยตรง เพื่อขอคำแนะนำเกี่ยวกับโครงสร้างของโปรเจกต์และทำการอัปเดตไฟล์ให้ตรงกับคลังหลักจนสามารถทำงานต่อได้สำเร็จ
